This chapter presents the design and implementation of the two Descheduler components developed in this research. Section 3.1 describes the defragmentation-focused \textit{ResourceDefragmentation} policy. Section 3.2 describes the Network-Aware Evictor and its topology- and latency-based cost models.

\section{ResourceDefragmentation Policy Design}
\label{sec:imbalanceindex_design}

\textit{ResourceDefragmentation} is a \textit{Balance} plugin that repairs scattered and poorly shaped CPU-memory allocation. It builds a request-based snapshot of worker Nodes, identifies Nodes that should be drained, predicts a feasible landing Node for every candidate Pod, and evicts the Pod whose predicted destination has the best projected CPU-memory bin score. The kube-scheduler remains responsible for final binding.

The experimental configuration explicitly uses declared Pod requests. This keeps Node assessment, feasibility checks, and target prediction aligned with the admission model used by kube-scheduler.

\subsection{Node-State Metrics}
\label{sec:defrag_node_metrics}

For Node \(n\), let \(u_{n,c}\) and \(u_{n,m}\) denote normalized requested CPU and memory:

\begin{equation}
    u_{n,c} = \frac{R_{n,c}}{C_{n,c}},
    \qquad
    u_{n,m} = \frac{R_{n,m}}{C_{n,m}},
\end{equation}

where \(R\) is the sum of Pod requests and \(C\) is allocatable capacity. Average utilization represents density:

\begin{equation}
    U_n^{avg} = \frac{u_{n,c} + u_{n,m}}{2}.
    \label{eq:defrag_avg_utilization}
\end{equation}

The bin score combines density and dimensional balance:

\begin{equation}
    B_n =
    \frac{
        \frac{u_{n,c}+u_{n,m}}{2}
        \left(1-\left|u_{n,c}-u_{n,m}\right|\right)
    }{2}.
    \label{eq:defrag_bin_score}
\end{equation}

A dense and balanced Node has a high \(B_n\), whereas an empty or strongly skewed Node has a lower score. The signed Resource Imbalance Index identifies the skew direction:

\begin{equation}
    RII_n = u_{n,c} - u_{n,m}.
    \label{eq:rii_node}
\end{equation}

The Free-Space Index represents joint CPU-memory headroom:

\begin{equation}
    FSI_n = (1-u_{n,c})(1-u_{n,m}).
    \label{eq:defrag_fsi}
\end{equation}

\subsection{Drain-Candidate Detection and Ordering}
\label{sec:defrag_drain_candidates}

A worker Node with at least one evictable Pod becomes a drain candidate when either its average utilization or its bin score is below the configured drain threshold \(\tau\):

\begin{equation}
    U_n^{avg} < \tau
    \quad \lor \quad
    B_n < \tau.
    \label{eq:defrag_drain_condition}
\end{equation}

The disjunction allows the policy to act on two relevant states. A lightly utilized but balanced Node can be drained to free infrastructure, while a more utilized but badly skewed Node can be drained to improve resource shape.

Drain candidates are processed in descending priority:

\begin{equation}
    P_n =
    0.5\left|RII_n\right|
    +
    0.5\frac{1}{FSI_n+\epsilon}.
    \label{eq:priority_index}
\end{equation}

The first term emphasizes dimensional skew. The second term emphasizes a narrow jointly usable free-space region. The constant \(\epsilon\) prevents division by zero.

\subsection{Predicted Scheduler Target}
\label{sec:defrag_target_prediction}

For every evictable Pod \(p\) on source Node \(s\), the plugin evaluates non-origin worker Nodes. A Node \(t\) is feasible only when:

\begin{itemize}
    \item the Node is schedulable and the Pod tolerates its taints;
    \item the Pod's CPU and memory requests fit;
    \item projected CPU and memory utilization do not exceed the configured target ceiling; and
    \item the target's minimum CPU-memory utilization is not lower than that of the source.
\end{itemize}

The final condition is the pack-upward guard:

\begin{equation}
    \min(u_{t,c},u_{t,m}) \geq \min(u_{s,c},u_{s,m}).
    \label{eq:defrag_pack_upward}
\end{equation}

It prevents an eviction from merely moving a Pod to an emptier Node. Among feasible Nodes, the predicted target is the Node with the highest projected bin score after receiving \(p\).

\subsection{Predicted-Target Eviction-Candidate Selection}
\label{sec:defrag_victim_selection}

For candidate Pod \(p\), let \(t_p\) be its predicted target and \(B'_{t_p}\) the target bin score after placement. The selected eviction candidate is:

\begin{equation}
    p^\star = \arg\max_{p} B'_{t_p}.
    \label{eq:defrag_selector}
\end{equation}

This is the only eviction-candidate selection criterion. It directly prefers a movement that combines complementary CPU-memory shapes at the expected destination. Candidates without a feasible target are excluded. After a successful eviction, the local source and predicted-target snapshots are updated before the next decision. Predicted targets are marked as bins so the same pass does not subsequently drain a Node that has just received packed workload.
