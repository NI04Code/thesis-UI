Dynamic Kubernetes clusters accumulate fragmented CPU and memory capacity as workloads change over time. Although aggregate capacity may remain sufficient, incompatible resource shapes can strand capacity across Nodes and prevent efficient workload placement. This study presents a multi-level Kubernetes Descheduler architecture that combines a \textit{ResourceDefragmentation} policy with network-aware eviction filtering. The resource policy assesses Nodes using average utilization, a CPU-memory bin score, the Resource Imbalance Index (RII), and the Free-Space Index (FSI). It predicts feasible scheduler targets before eviction and selects a victim using the projected CPU-memory bin score of the predicted landing Node. Eviction is allowed only when the target remains below a utilization ceiling and is at least as packed as the source. Across five request-based scenarios, the policy reclaimed four Nodes in the underutilized scenario, three Nodes in the fragmented scenario, and reduced the cluster stranding score from \(1.338\) to \(0.123\) in the heterogeneous-drain scenario using two evictions. All runs converged without leaving Pods Pending. The network-aware layer further blocks movements that would degrade communication locality. These results show that predicted-target balance can produce effective defragmentation while retaining explicit feasibility and network-locality safeguards.
